git-show-index(1)
=================

NOM
---
git-show-index - Affiche l'index des archives empaquetées


SYNOPSIS
--------
[verse]
'git show-index' [--object-format=<algorithme-d-empreinte>] < <fichier-d-index-de-paquet>


DESCRIPTION
-----------
Lire le fichier `.idx` pour un fichier paquet Git (créé avec linkgit:git-pack-objects[1] ou linkgit:git-index-pack[1]) depuis l'entrée standard, et décharger son contenu. La sortie comprend un objet par ligne, chaque ligne contenant deux ou trois colonnes séparées par espace :

  - la première colonne est le décalage en octets de l'objet dans le fichier paquet correspondant

  - la seconde colonne est l'id de l'objet

  - si la version d'index est 2 ou supérieure, la troisième colonne contient le CRC32 des données de l'objet

Les objets sont affichés dans l'ordre dans lequel ils sont trouvés dans le fichier index, qui devrait être (dans un fichier correctement construit) trié par id d'objet.

Notez que vous pouvez obtenir plus d'informations sur un fichier paquet en appelant linkgit:git-verify-pack[1]. Cependant, comme cette commande ne considère que le fichier index lui-même, il est à la fois plus rapide et plus flexible.

OPTIONS
-------

--object-format=<algorithme-empreinte>::
	Spécifier le format d'objet donné (algorithme d'empreinte) pour le fichier d'index. Les valeurs valides sont 'sha1' et (si activé) 'sha256'. La valeur par défaut est l'algorithme pour le dépôt actuel (défini par `extensions.objectFormat`), ou 'sha1' si aucune valeur n'est définie ou en dehors d'un dépôt.

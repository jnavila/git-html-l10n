`--signoff`::
`--no-signoff`::
	Agrega un colofón `Signed-off by` por el confirmante al final del mensaje de registro de confirmación. El significado de una aprobación depende del proyecto al que se confirma. Por ejemplo, puede certificar que el confirmante tiene derecho a enviar el trabajo bajo la licencia del proyecto, o acepta alguna representación del contribuyente, como un certificado de origen del desarrollador. (Ver https://developercertificate.org para el usado por los proyectos del kernel de Linux y Git.) Consulte la documentación o los líderes del proyecto al que contribuye para entender cómo se usan las aprobaciones en ese proyecto.
+
La opción `--no-signoff` puede usarse para contraordenar una opción `--signoff` previa en la línea de comandos.
+
Git no tiene (ni tendrá) una variable de configuración para habilitar la opción de línea de comando `--signoff` predeterminadamente; ver la entrada `commit.signoff` en linkgit:gitfaq[7] para más detalles.

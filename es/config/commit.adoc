`commit.cleanup`::
	Esta configuración anula el predeterminado de la opción `--cleanup` en `git commit`. {see-git-commit} Cambiar el predeterminado puede ser útil cuando quieres siempre mantener líneas que comiencen con el caracter (`core.commentChar`, predeterminado a`#`) en tu mensaje de bitácora, en cuyo caso harías `git config commit.cleanup whitespace` (nota que, si haces esto, tendrás que quitar por ti mismo las líneas de ayuda que comienzan con el caracter de comentario en la plantilla de bitácora de confirmación).

`commit.gpgSign`::
	Un booleano para especificar si todas las confirmaciones deben ser firmadas con GPG. Usar esta opción cuando se hagan operaciones como rebase que puedan resultar en un número elevado de confirmaciones a firmar. Puede ser conveniente usar un agente para evitar teclear tu contraseña GPG varias veces.

`commit.status`::
	Un booleano para habilitar/deshabilitar la inclusión de información de estado en la plantilla de mensaje de confirmación cuando se usa un editor para preparar el mensaje de confirmación. Es predeterminado a `true`.

`commit.template`::
	Especifica el nombre de ruta de un fichero que será usado como plantilla de nuevos mensajes de confirmación.

`commit.verbose`::
	Un booleano o entero para especificar el nivel de verbosidad con `git commit`. {see-git-commit}

`branch.autoSetupMerge`::
	Anger för `git branch`, `git switch` och `git checkout` att ställa in nya grenar så att linkgit:git-pull[1] sammanslås korrekt från startpunktsgrenen. Observera att även om detta alternativ inte är angivet kan detta beteende väljas per gren med hjälp av alternativen `--track` och `--no-track`. Standardinställningen för detta alternativ är `true`. De giltiga inställningarna är:
`false`;; ingen automatisk inställning görs
`true`;; automatisk inställning görs när startpunkten är en fjärrspårad gren
`always`;; automatisk inställning görs när startpunkten är antingen en lokal gren eller en fjärrspårad gren
`inherit`;; Om startpunkten har en spårningskonfiguration, kopieras den till den nya grenen
`simple`;; automatisk inställning görs endast när startpunkten är en fjärrspårad gren och den nya grenen har samma namn som fjärrgrenen.

`branch.autoSetupRebase`::
	När en ny gren skapas med `git branch`, `git switch` eller `git checkout` och spårar en annan gren, anger variabeln att Git ska ställa in pull till att ombasera i stället för att sammanfoga (se `branch.<name>.rebase`). Giltiga inställningar är:
`never`;; ombasera sätts aldrig automatiskt till true.
`local`;; Visa både fjärrspårade grenar och lokala grenar.
`remote`;; rebase är satt till sant för spårade grenar av fjärrspårande grenar.
`always`;; ombasera kommer att sättas till true för alla spårningsgrenar.

+
Se `branch.autoSetupMerge` för information om hur man konfigurerar en gren för att spåra en annan gren. Standardinställningen är `never`.

`branch.sort`::
	Denna variabel styr sorteringsordningen för grenar när de visas av linkgit:git-branch[1]. Utan alternativet `--sort=<värde>` kommer värdet för denna variabel att användas som standard. Se fältnamnen i linkgit:git-for-each-ref[1] för giltiga värden.

`branch.<namn>.mergeOptions`::
	När grenen är _<namn>_ anger den för `git fetch` och `git push` vilken fjärr som ska hämtas från eller skickas till. Fjärren som skickas till kan åsidosättas med `remote.pushDefault` (för alla grenar). Fjärren som skickas till, för den aktuella grenen, kan ytterligare åsidosättas av `branch.<namn>.pushRemote`. Om ingen fjärr är konfigurerad, eller om ingen gren är aktiv och det finns mer än en fjärr definierad i kodförrådet, används som standard `origin` för hämtning och `remote.pushDefault` för push:ing. Dessutom är `.` (en punkt) det aktuella lokala kodförrådet (ett punkt-kodförråd), se `branch.<namn>.merge`s sista anmärkning nedan.

`branch.<namn>.mergeOptions`::
	För grenen _<name>_ åsidosätter den `branch.<name>.remote` för push. Den åsidosätter också `remote.pushDefault` för push från grenen _<name>_. När hämtning sker från en plats (t.ex. uppströmsen) och push till en annan plats (t.ex. ett eget publiceringskodförråd) kan `remote.pushDefault` för att ange vilken fjärr som ska användas för push för alla grenar, och använda detta alternativ för att åsidosätta det för en specifik gren.

`branch.<namn>.mergeOptions`::
	Definierar, tillsammans med `branch.<name>.remote`, uppströmsgrenen för den givna grenen. Den anger för `git fetch`/`git pull`/`git rebase` vilken gren som ska sammanfogas och kan även påverka `git push` (se `push.default`). För grenen _<name>_ anger den för `git fetch` vilken standardrefspec som ska markeras för sammanfogning i `FETCH_HEAD`. Värdet hanteras som fjärrdelen av en refspec och måste matcha en ref som hämtas från fjärren som anges av `branch.<name>.remote`. Sammanfogningsinformationen används av `git pull` (som först anropar `git fetch`) för att slå upp standardgrenen för sammanfogning. Utan detta alternativ sammanfogar `git pull` som standard den första hämtade refspecen. Ange flera värden för att få en bläckfisksammanslagning. Om `git pull` ska konfigureras så att den sammanfogar till _<name>_ från en annan gren i det lokala kodförrådet, kan `branch.<name>.merge` peka på önskad gren och den relativa sökvägsinställningen `.` (en punkt) för `branch.<name>.remote`.

`branch.<namn>.mergeOptions`::
	Sätter standardalternativ för sammanslagning till grenen _<name>_. Syntaxen och de alternativ som stöds är desamma som i linkgit:git-merge[1], men alternativvärden som innehåller blanktecken stöds för närvarande inte.

`branch.<namn>.mergeOptions`::
	När det är sant, ombasera grenen _<namn>_ ovanpå den hämtade grenen, i stället för att sammanfoga standardgrenen från standardfjärren när `git pull` körs. Se `pull.rebase` för att göra detta på ett icke-grenspecifikt sätt.
+
När `merges` (eller bara `m`) används skickas alternativet `--rebase-merges` till `git rebase` så att lokala sammanslagningsincheckningar inkluderas i rebasen (se linkgit:git-rebase[1] för detaljer).
+
När värdet är `interactive` (eller bara `i`) körs ombaseringen i interaktivt läge.
+
*OBS*: detta är en potentiellt farlig operation; använd den *inte* om du inte förstår konsekvenserna (se linkgit:git-rebase[1] för mer information).

`branch.<namn>.mergeOptions`::
	Grenbeskrivning, kan redigeras med `git branch --edit-description`. Grenbeskrivningen läggs automatiskt till i `format-patch` följebrevet eller `request-pull` sammanfattningen.

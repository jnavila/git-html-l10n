git-remote-fd(1)
================

NAMN
----
git-remote-fd - Reflektera smart transportström tillbaka till anroparen

SYNOPSIS
--------
"fd::<infd>[,<outfd>][/<vad-som-helst>]" (som URL)

BESKRIVNING
-----------
Denna hjälpfunktion använder specificerade filbeskrivningar för att ansluta till en fjärransluten Git-server. Detta är inte avsett för slutanvändare utan för program och skript som anropar git fetch, push eller archive.

Om endast <infd> anges antas det vara en dubbelriktad socket ansluten till en fjärransluten Git-server (git-upload-pack, git-receive-pack eller git-upload-archive). Om både <infd> och <outfd> anges antas de vara pipes anslutna till en fjärransluten Git-server (<infd> är den inkommande pipen och <outfd> är den utgående pipen).

Det antas att alla handskakningsprocedurer redan har slutförts (som att skicka en serviceförfrågan för git://) innan denna hjälpprogram startas.

<vad-som-helst> kan vara vilken sträng som helst. Den ignoreras. Den är avsedd att ge information till användaren i URL:en om URL:en visas i något sammanhang.

MILJÖVARIABLER
--------------
GIT_TRANSLOOP_DEBUG::
	Om angivet skrivs felsökningsinformation om olika läsningar/skrivningar ut.

EXEMPEL
-------
`git fetch fd::17 master`::
	Hämta master, med hjälp av fildeskriptor #17 för att kommunicera med git-upload-pack.

`git fetch fd::17/foo master`::
	Samma som ovan.

`git push fd::7,8 master (som URL)`::
	Push-master, med fildeskriptor #7 för att läsa data från git-receive-pack och fildeskriptor #8 för att skriva data till samma tjänst.

`git push fd::7,8/bar master`::
	Samma som ovan.

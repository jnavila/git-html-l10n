git-fmt-merge-msg(1)
====================

NOME
----
git-fmt-merge-msg - Gera uma mensagem de commit da mesclagem


SINOPSE
-------
[verse]
'git fmt-merge-msg' [-m <mensagem>] [--log[=<n>] | --no-log]
'git fmt-merge-msg' [-m <mensagem>] [--log[=<n>] | --no-log] -F <arquivo>

DESCRIÇÃO
---------
Pega a lista de objetos mesclados no stdin e gera uma mensagem adequada para
o commit, que foi mesclado, geralmente ser utilizada como
'<mensagem-da-mesclagem>' que é um argumento do comando 'git merge'.

Primeiramente, este comando é destinado para uso interno com scripts que
invocam automaticamente o 'git merge'.

OPÇÕES
------

--log[=<n>]::
	Além dos nomes das ramificações, preencha a mensagem do registro log com
	descrições de uma linha dos commits que estão sendo mesclados atualmente.
	Serão utilizadas no máximo <n> mesclagens dos commits de cada origem (20
	caso <n> seja omitido).  Isso substitui a variável da configuração
	`merge.log`.

--no-log::
	Não liste os descritivos de uma linha dos commits atuais que estão sendo
	mescladas.

--[no-]summary::
	É um sinônimos para `--log` e `--no-log`; estas opções foram descontinuadas
	e serão removidas no futuro.

-m <mensagem>::
--message <mensagem>::
	Para a primeira linha da mensagem do registro log utilize `<mensagem>` em
	vez dos nomes das ramificações.  Para usar com `--log`.

-F <arquivo>::
--file <arquivo>::
	Pegue a lista dos objetos mesclados de <arquivo> em vez do stdin.

CONFIGURAÇÃO
------------
include::config/fmt-merge-msg.txt[]

merge.summary::
	É um sinônimo para `merge.log`; estas opções foram descontinuadas e serão
	removidas no futuro.

EXEMPLOS
--------

---------
$ git fetch origin master
$ git fmt-merge-msg --log <$GIT_DIR/FETCH_HEAD
---------

Imprima uma mensagem do registro log descrevendo uma mesclagem do ramo
"master" no ramo remoto "origin".

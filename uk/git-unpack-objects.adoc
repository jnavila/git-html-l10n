git-unpack-objects(1)
=====================

НАЗВА
-----
git-unpack-objects — Розпакувати обʼєкти з упакованого архіву


СИНОПСИС
--------
[verse]
'git unpack-objects' [-n] [-q] [-r] [--strict]


ОПИС
----
Зчитує упакований архів (.pack) зі стандартного вводу, розпаковуючи обʼєкти, що містяться в ньому, та записуючи їх у репозиторій у "вільному" форматі (один обʼєкт на файл).

Обʼєкти, які вже існують у репозиторії, *не* будуть розпаковані з пакетного файлу. Тому нічого не буде розпаковано, якщо ви використаєте цю команду для пакетного файлу, який існує в цільовому репозиторії.

Дивіться linkgit:git-repack[1] для отримання інформації про опції створення нових пакетів та заміни наявних.

ОПЦІЇ
-----
-n::
Пробний запуск. Перевірте файл упаковки без фактичного розпакування обʼєктів.

-q::
	Команда зазвичай показує прогрес у відсотках. Цей прапорець його пригнічує.

-r::
	Під час розпакування пошкодженого пакетного файлу команда завершує роботу при першому пошкодженні. Цей прапорець вказує їй продовжувати роботу та докласти всіх зусиль для відновлення якомога більшої кількості обʼєктів.

--strict::
	Не створювати обʼєкти з пошкодженим вмістом або посиланнями.

--max-input-size=<size>::
	Припиняти роботу, якщо пакунок більший за <розмір>.

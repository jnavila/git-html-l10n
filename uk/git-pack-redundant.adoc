git-pack-redundant(1)
=====================

НАЗВА
-----
git-pack-redundant — Пошук надлишкових pack-файлів


СИНОПСИС
--------
[verse]
'git pack-redundant' [--verbose] [--alt-odb] (--all | <pack-filename>...)

ПОПЕРЕДЖЕННЯ
------------
Команда `git pack-redundant` є застарілою і планується її видалення в майбутніх версіях Git. Оскільки вона може видаляти лише повторювані пакунки цілком, а не окремі дублікати об’єктів, зазвичай вона не є ефективним інструментом для зменшення розміру репозиторію. Для цього краще використовувати команду `git gc`, яка перемістить об’єкти до нового пакунка, видаливши дублікати.

Запуск `pack-redundant` без прапорця `--i-still-use-this` не спрацює в цьому випуску. Якщо ви вважаєте, що у вас є варіант використання, для якого `pack-redundant` краще підходить, і заперечуєте проти цього видалення, будь ласка,напишіть у список розсилки Git за адресою git@vger.kernel.org. Більше інформації про список розсилки доступно за адресою https://git-scm.com/community.

ОПИС
----
Ця програма обчислює, які пакунки у вашому репозиторії є надлишковими. Вивід підходить для перенаправлення до `xargs rm`, якщо ви знаходитесь у кореневій теці репозиторію.

Команда 'git pack-redundant' приймає список обʼєктів у стандартний ввід. Будь-які надані обʼєкти будуть ігноруватися під час перевірки того, які пакунки є обовʼязковими. Це робить наступну команду корисною, якщо потрібно видалити пакунки, що містять недоступні обʼєкти.

git fsck --full --unreachable | cut -d ' ' -f3 | \ git pack-redundant --all | xargs rm

ОПЦІЇ
-----


--all::
	Обробляти всі пакунки. Будь-які імена файлів у командному рядку ігноруються.

--alt-odb::
	Не вимагати, щоб обʼєкти, присутні в пакунках з тек альтернативної бази даних обʼєктів (odb), були присутні в локальних пакунках.

--verbose::
	Виводить деяку статистику в stderr. Має невелике зниження продуктивності.

git-diff-files(1)
=================

НАЗВА
-----
git-diff-files — Порівняння файлів у робочому дереві та індексі


СИНОПСИС
--------
[verse]
'git diff-files' [-q] [-0 | -1 | -2 | -3 | -c | --cc] [<common-diff-options>] [<path>...]

ОПИС
----
Порівнює файли в робочому дереві та індексі. Якщо вказано шляхи, порівнюються лише файли за вказаними шляхами. В іншому випадку порівнюються всі записи в індексі. Формат виводу такий самий, як у команд 'git diff-index' та 'git diff-tree'.

ОПЦІЇ
-----
include::diff-options.adoc[]

-1 --base::
-2 --ours::
-3 --theirs::
-0::
	Різниця з "базовою" версією, "нашою гілкою" або "їхньою гілкою" відповідно. З цими опціями різниці для обʼєднаних записів не показуються.
+
Зазвичай виконується порівняння з нашою гілкою (-2) та правильно встановленими шляхами. Можна вказати опцію -0, щоб приховати вивід diff для не обʼєднаних записів і показувати лише «Unmerged».

-c::
--cc::
	Ця команда порівнює satge 2 (нашу гілку), satge 3 (їхню гілку) та файл робочого дерева і виводить об’єднаний diff, подібно до того, як команда «diff-tree» показує коміт злиття з цими прапорцями.

-q::
	Не виводити повідомлення навіть для файлів, яких не існує

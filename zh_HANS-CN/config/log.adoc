`log.abbrevCommit`::
	如果为 `true`，则让
ifndef::with-breaking-changes[]
	linkgit:git-log[1]、linkgit:git-show[1] 和
	linkgit:git-whatchanged[1]
endif::with-breaking-changes[]
ifdef::with-breaking-changes[]
	linkgit:git-log[1] 和 linkgit:git-show[1]
endif::with-breaking-changes[]
	默认采用 `--abbrev-commit`。你可以使用
	`--no-abbrev-commit` 覆盖此选项。

`log.date`::
	为 `log` 命令设置默认的日期时间模式。为 `log.date` 设置值，类似于为 `git log` 使用 `--date` 选项。详见 linkgit:git-log[1]。
+
如果格式设置为 "auto:foo"，且正在使用分页器，则日期格式将使用 "foo"。否则将使用 "default"。

`log.decorate`::
	打印 `log` 命令显示的各个提交的 ref 名称。可取值包括：
+
--
`short`;; 不打印 ref 名称前缀 `refs/heads/`、`refs/tags/` 和 `refs/remotes/`。
`full`;; 打印完整的 ref 名称（包括前缀）。
`auto`;; 如果输出将发往终端，则按给出 `short` 时的方式显示 ref 名称；否则不显示 ref 名称。
--
+
和 linkgit:git-log[1] 的 `--decorate` 选项相似。

`log.initialDecorationSet`::
	默认情况下，`git log` 只会显示某些已知 ref 命名空间的修饰信息。如果指定为 'all'，则将所有 ref 都显示为修饰信息。

`log.excludeDecoration`::
	从日志修饰信息中排除指定模式。这类似于 `--decorate-refs-exclude` 命令行选项，但该配置项可被 `--decorate-refs` 选项覆盖。

`log.diffMerges`::
	设置在指定 `--diff-merges=on` 时使用的 diff 格式。详见 linkgit:git-log[1] 中的 `--diff-merges`。默认值为 `separate`。

`log.follow`::
	如果为 `true`，则当给出单个 <路径> 时，`git log` 的行为就如同使用了 `--follow` 选项。这与 `--follow` 有相同的限制，即它不能用于跟踪多个文件，且在非线性历史上效果不佳。

`log.graphColors`::
	一个用逗号分隔的颜色列表，可用于在 `git log --graph` 中绘制历史线。

`log.showRoot`::
	如果为 true，则会将初始提交显示为一个重大的创建事件。这等价于与空树进行 diff。像 linkgit:git-log[1] 或 linkgit:git-whatchanged[1] 这样通常会隐藏根提交的工具，此时也会显示它。默认值为 true。

`log.showSignature`::
	如果为真，则使 linkgit:git-log[1]、linkgit:git-show[1] 和 linkgit:git-whatchanged[1] 默认应用 `--show-signature` 选项。

`log.mailmap`::
	如果为真，则使 linkgit:git-log[1]、linkgit:git-show[1] 和 linkgit:git-whatchanged[1] 默认应用 `--use-mailmap` 选项，否则使其默认应用 `--no-use-mailmap` 选项。默认为真。

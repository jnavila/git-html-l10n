git-credential-cache{litdd}daemon(1)
====================================

НАЗВАНИЕ
--------
git-credential-cache--daemon — временное хранение учётных данных пользователя в памяти

ОБЗОР
-----
[verse]
'git credential-cache{litdd}daemon' [--debug] <путь-сокета>

ОПИСАНИЕ
--------

NOTE: Вероятно, вы не захотите вызывать эту команду самостоятельно; она запускается автоматически, когда вы используете linkgit:git-credential-cache[1].

Эта команда прослушивает сокет домена Unix, указанный `<путь-сокета>`, для клиентов `git-credential-cache`. Клиенты могут сохранять и извлекать учётные данные. Каждые учётные данные удерживаются в течение тайм-аута, указанного клиентом; как только учётные данные не удерживаются, демон завершает работу.

Если указан параметр `--debug`, демон не закрывает свой поток stderr и может выводить в него дополнительную диагностику, даже после того, как начал прослушивать клиентов.

git-pack-redundant(1)
=====================

НАЗВАНИЕ
--------
git-pack-redundant — Поиск избыточных pack-файлов


ОБЗОР
-----
[verse]
'git pack-redundant' [--verbose] [--alt-odb] (--all | <имя-файла-пакета>...)

ПРЕДУПРЕЖДЕНИЕ
--------------
`git pack-redundant` устарел и запланирован к удалению в одной из будущих версий Git. Поскольку он может удалять только целые повторяющиеся pack-файлы, а не отдельные повторяющиеся объекты, он, как правило, не является полезным инструментом для уменьшения размера репозитория. Вам лучше использовать для этого `git gc`, который поместит объекты в новый pack-файл, удаляя дубликаты.

Запуск `pack-redundant` без флага `--i-still-use-this` в этом выпуске завершится ошибкой. Если вы считаете, что у вас есть вариант использования, для которого `pack-redundant` подходит лучше, и вы возражаете против этого удаления, пожалуйста, свяжитесь со списком рассылки Git по адресу git@vger.kernel.org. Дополнительная информация о списке доступна по адресу https://git-scm.com/community.

ОПИСАНИЕ
--------
Эта программа вычисляет, какие pack-файлы в вашем репозитории являются избыточными. Вывод подходит для передачи в `xargs rm`, если вы находитесь в корне репозитория.

'git pack-redundant' принимает список объектов на стандартном вводе. Любые заданные объекты будут игнорироваться при проверке того, какие pack-файлы необходимы. Это делает следующую команду полезной, когда требуется удалить pack-файлы, содержащие недостижимые объекты.

git fsck --full --unreachable | cut -d ' ' -f3 | \ git pack-redundant --all | xargs rm

ПАРАМЕТРЫ
---------


--all::
	Обрабатывает все pack-файлы. Любые имена файлов в командной строке игнорируются.

--alt-odb::
	Не требовать присутствия объектов из pack-файлов в каталогах дополнительной базы данных объектов (odb) в локальных pack-файлах.

--verbose::
	Выводит некоторую статистику в stderr. Имеет небольшое снижение производительности.

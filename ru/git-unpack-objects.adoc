git-unpack-objects(1)
=====================

НАЗВАНИЕ
--------
git-unpack-objects — распаковка объектов из упакованного архива


ОБЗОР
-----
[verse]
'git unpack-objects' [-n] [-q] [-r] [--strict]


ОПИСАНИЕ
--------
Читает упакованный архив (.pack) из стандартного ввода, распаковывая содержащиеся в нём объекты и записывая их в репозиторий в "несжатом" формате (один объект на файл).

Объекты, которые уже существуют в репозитории, *не будут* распакованы из pack-файла. Поэтому ничего не будет распаковано, если вы используете эту команду для pack-файла, который существует в целевом репозитории.

См. linkgit:git-repack[1] для получения параметров создания новых пакетов и замены существующих.

ПАРАМЕТРЫ
---------
-n::
Пробный запуск. Проверить pack-файл без фактической распаковки объектов.

-q::
	Команда обычно показывает процент выполнения. Этот флаг подавляет его.

-r::
	При распаковке повреждённого pack-файла команда завершается критической ошибкой при первом же повреждении. Этот флаг указывает ей продолжать и приложить все усилия для восстановления как можно большего количества объектов.

--strict::
	Не записывать объекты с повреждённым содержимым или связями.

--max-input-size=<size>::
	Завершиться критической ошибкой, если пакет больше <размера>.

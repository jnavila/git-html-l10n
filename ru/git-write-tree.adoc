git-write-tree(1)
=================

НАЗВАНИЕ
--------
git-write-tree - Создание объекта дерева из текущего индекса


ОБЗОР
-----
[synopsis]
'git write-tree' [--missing-ok] [--prefix=<префикс>/]

ОПИСАНИЕ
--------
Создаёт объект-дерево, используя текущий индекс. Имя нового объекта-дерева выводится в стандартный поток вывода.

Индекс должен быть в полностью слитом состоянии.

Концептуально, `git write-tree` синхронизирует текущее содержимое индекса в набор файлов деревьев. Чтобы это соответствовало тому, что на самом деле находится в вашем каталоге прямо сейчас, перед выполнением `git write-tree` вам нужно было выполнить фазу `git update-index`.


ПАРАМЕТРЫ
---------
`--missing-ok`::
	Обычно `git write-tree` гарантирует, что объекты, на которые ссылается каталог, существуют в базе данных объектов. Этот параметр отключает эту проверку.

`--prefix=<префикс>/`::
	Записывает объект-дерево, представляющий подкаталог `<префикс>`. Это можно использовать для записи объекта-дерева для подпроекта, находящегося в указанном подкаталоге.
